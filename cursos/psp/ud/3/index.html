3. Programación en red

Actividades de iniciación

JS01.java Crea un servidor que escuche y atienda a un solo cliente por el puerto 10001. Una vez aceptada la conexión, le enviará la cadena ¡Hola, mundo! y el servidor finalizará.
JC01.java Crea un cliente que se conecte a un servidor por el puerto 10001. Una vez conectado, recibirá la cadena ¡Hola, mundo! desde el servidor y el cliente finalizará.

JS02.java Crea un servidor que escuche y atienda a un solo cliente por el puerto 10002. Una vez aceptada la conexión, le enviará cincuenta enteros aleatorios del 0 al 10 (ambos incluidos) y el servidor finalizará.
JC02.java Crea un cliente que se conecte a un servidor por el puerto 10002. Una vez conectado, recibirá cincuenta enteros aleatorios del 0 al 10 (ambos incluidos) desde el servidor y el cliente finalizará.

JS03.java Crea un servidor que escuche y atienda a un solo cliente por el puerto 10003. Una vez aceptada la conexión, esperará un mensaje (cadena de caracteres) del cliente y le enviará el mismo mensaje. Si recibe el mensaje Adiós., el servidor finalizará.
JC03.java Crea un cliente que se conecte a un servidor por el puerto 10003. Una vez conectado, esperará un mensaje (cadena de caracteres) del usuario y enviará dicho mensaje al servidor. El cliente recibirá del servidor el mismo mensaje. Si el usuario introduce Adiós., el cliente finalizará.

JS04.java Crea un servidor que escuche y atienda a varios clientes por el puerto 10004. Una vez aceptada la conexión de un cliente, esperará un mensaje (cadena de caracteres) del cliente y le enviará el mismo mensaje.
JC04.java Crea un cliente que se conecte a un servidor por el puerto 10004. Una vez conectado, esperará un mensaje (cadena de caracteres) del usuario y enviará dicho mensaje al servidor. El cliente recibirá del servidor el mismo mensaje. Si el usuario introduce Adiós., el cliente finalizará.

Actividades

Crea las siguientes aplicaciones cliente-servidor (un servidor podrá atender a varios clientes a la vez):

JC11.java - JS11.java (puerto 10011) El servidor generará para cada cliente un entero aleatorio entre 0 y 100 (ambos incluidos). El cliente solicitará al usuario un entero y lo enviará al servidor hasta que adivine el entero generado. El servidor indicará, para cada entero recibido por el cliente, si es mayor, menor o ha acertado. Si acierta, terminará la conexión del cliente y se le indicará en cuantos intentos lo ha conseguido.

JC12.java - JS12.java (puerto 10012) Crea un servidor de eco (como el ejercicio JC04.java - JS04.java) controlando el acceso de los clientes al servidor mediante un sistema de usuario y clave. El servidor almacenará, mediante arrays de Strings, los usuarios grego, alba y pedro y las contraseñas grego1234, alba1234 y pedro1234 respectivamente. Una vez el usuario se ha conectado exitosamente, podrá salir con Adiós..

Realiza dos proyectos Java cliente/servidor usando el puerto 2244 para - En cada cliente muestra "Introduzca usuario:", lee un usuario del teclado, muestra "Introduzca clave:", lee una clave del teclado, conéctate al servidor y envía el usuario y la clave. Si recibes el mensaje "0 Conexión rechazada" acabas. Si recibes el mensaje "1 Conexión aceptada" métete en un bucle que muestre "Introduzca mensaje o 'fin' para terminar:", lee un mensaje del teclado y envíalo al servidor (cuando teclee "fin" termina el bucle). - El servidor debe tener dos arrays: String[] usuarios={"abcde","juan","pablo"}; String[] claves={"pepeclave","clavejuan","unaclave"}; Cuando el servidor detecta una conexión de un cliente, crea un hilo para ese cliente, recibe un usuario y una clave. Si el usuario o la clave no son correctas envíale un mensaje "0 Conexión rechazada" y acaba el hilo. Si el usuario y la clave son correctas envíale un mensaje "1 Conexión aceptada" y métete en un bucle recibiendo mensajes del cliente y mostrándolos hasta recibir el mensaje "fin". ============= EJERCICIO 2 ============= El objetivo del ejercicio es crear una aplicación cliente/servidor que permita el envío de ficheros al cliente. Para ello, el cliente se conectará al servidor por el puerto 1500 y le solicitará el nombre de un fichero del servidor. Si el fichero existe, el servidor, le enviará el fichero al cliente y éste lo mostrará por pantalla. Si el fichero no existe, el servidor le enviará al cliente un mensaje de error. Una vez que el cliente ha mostrado el fichero se finalizará la conexión.
