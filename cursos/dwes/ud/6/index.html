6. Aplicaciones web dinámicas con Laravel, Bootstrap y Vue

Actualmente, un desarrollador web full stack usa frameworks web tanto para front end (Bootstrap, Angular, Vue, etc.) como back end (Laravel, etc.). La opción más popular actualmente son las aplicaciones web AJAX, es decir, aplicaciones web cuya interacción con el servidor es asíncrona. En este ejemplo se puede observar cómo se consultan datos a una base de datos de manera asíncrona.

Vue es un framework JavaScript que permite desarrollar aplicaciones front end e interactuar con el back end de manera asíncrona.

1. Instala Npm

Para esta actividad, es necesario usar Npm, un sistema de gestión de paquetes de JavaScript. Npm es el equivalente de Composer para PHP. Npm se distribuye con Node.js, por tanto:

  1. Instala Node.js:
    • Windows: mediante el instalador (versión LTS).
    • Debian/Ubuntu:
      • Instala las dependencias con sudo apt-get install gcc g++ make y después entra como root (su root).
      • Ejecuta curl -sL https://deb.nodesource.com/setup_12.x | bash - y después apt-get install -y nodejs.
      • Sal de root con exit.
  2. Comprueba que funciona Node.js y Npm en la terminal ejecutando node -v y npm -v respectivamente en Windows o en Debian/Ubuntu.

2. Crea un proyecto en Laravel e instala en él Bootstrap y Vue.js con Laravel UI

Laravel te facilita instalar y configurar automáticamente frontends como Bootstrap y Vue. Para ello, consulta su documentación que indica los siguientes pasos:

  1. Crea un proyecto en Laravel: composer create-project --prefer-dist laravel/laravel ejemplo_vue.
  2. Entra en el proyecto e instala la dependencia Laravel UI: composer require laravel/ui.
  3. Prepara la dependencia de Bootstrap y crea una estructura de autenticación con php artisan ui bootstrap --auth.
  4. Prepara la dependencia de Vue.js y crea una estructura de autenticación con php artisan ui vue --auth. Di sí a todo.
  5. Instala las dependencias preparadas con npm install && npm run dev.
  6. Ejecuta php artisan serve y prueba con http://localhost:8000. Para probar la autenticación configura la base de datos antes.
