3. PHP orientado a objetos e introducción a frameworks web MVC

Actividades

Crea una página inicio.html que enlazca las siguientes actividades a resolver:

1. Crea una página de ejemplo con autenticación HTTP básica. A continuación, en dicha página, responde: ¿por qué no es seguro esta autenticación? ¿Cómo podría mejorarse? Para responder, puedes ayudarte con este material.

2. Crea una página de ejemplo con autenticación HTTP hash/digest con el algoritmo Blowfish/Bcrypt usando crypt(). A continuación, en dicha página, responde: ¿qué es una función hash? ¿Por qué es tan importante en seguridad informática? ¿Cuál sería la autenticación HTTP más segura de todas finalmente? Para responder, puedes ayudarte con este material.

3. Crea una cesta de la compra de frutas y verduras y almacénala durante una hora usando cookies. El usuario elegirá una serie de frutas y verduras y mantendrá dichos alimentos escogidos durante una hora aunque cierre el navegador. Mediante setcookie(), sin autenticación ni sesiones, almacena dicha cesta y crea también opciones para modificar, eliminar y comprobar si existe dicha cookie. Puedes ayudarte de los materiales de Diego Lázaro y de W3Schools.

4. Crea un sitio web que resuelva los primeros cinco de estos ejercicios sobre clases. Se crearán cinco pestañas, en cada una se resolverá un ejercicio. Añade una pestaña más explicando, con capturas, cómo se depura en PHP y muestra un ejemplo.

5. Crea un balance sencillo de ingresos y gastos. Para obtener el balance, habrá que registrarse como usuario en una base de datos y después iniciar sesión; las contraseñas estarán almacenadas en hash mediante el algoritmo Bcrypt (usa password_hash y password_verify). Una vez iniciada la sesión correctamente, el usuario introducirá, mediante formularios, una serie de ingresos y gastos, pulsará en Generar y, a continuación, verá un informe similar a éste en PDF. Usa MPDF, instalálo con Composer y aloja esta aplicación web en Heroku y en GitHub.

6. Crea una aplicación web CRUD usando Laravel de una organización ficticia sobre una de estas temáticas:

  • Biblioteca: consulta y gestión de libros (ISBN, título, autor, editorial, edición, año).
  • Videoteca: consulta y gestión de películas (ISAN, título, director, guionista, productor, año).
  • Ludoteca: consulta y gestión de videojuegos (ISAN, título, desarrolladora, distribuidora, género, año).
  • Hemeroteca: consulta y gestión de periódicos (ISSN, título, director, editorial, idioma, frecuencia).

La aplicación web estará alojada en Heroku y tendrá como mínimo las siguientes pestañas:

  • Inicio: presentación de la organización y de la página web.
  • Información con las siguientes subpestañas:
    • MVC: explicación de lo que es un MVC.
    • Laravel: explicación de un framework web MVC como Laravel.
    • Instalación de Laravel: tutorial paso a paso de cómo instalar Laravel (si es propio, realizado por uno mismo, mejor).
    • Material: enlaces con material didáctico para aprender Laravel, por ejemplo éste.
  • Insertar: inserción de un producto nuevo en la organización.
  • Visualizar: visualización de todos los productos almacenados en la organización.
  • Editar: modificación de los campos de un producto determinado.
  • Borrar: eliminación de un producto almacenado en la organización.
  • Sobre nosotros: descripción y fines de la organización.
  • Contacto: datos de contacto para comunicarse con la organización.

Evaluación

El profesor descomprimirá el zip entregado, moverá la carpeta descomprimida a htdocs y abrirá inicio.html. Desde ahí accederá a las diferentes actividades para evaluar su funcionamiento: algunas enlazarán en local y otras a Heroku, según corresponda. Por último, el profesor analizará y evaluará el código.
