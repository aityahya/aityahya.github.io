2. Modelo Entidad-Relación y Modelo Relacional

Contenidos

  1. Creación de una base de datos:
    • Análisis:
      • Especificación de Requisitos de Software (ERS).
    • Diseño:
      • Fase 1 - Conceptual: modelo de datos Modelo Entidad-Relación (E-R).
      • Fase 2 - Lógico: modelo de base de datos Modelo Relacional.
      • Fase 3 - Físico: modelo físico (implementado en SQL) (se ve en la próxima unidad).

  2. Modelo Entidad-Relación (E-R):
    • Entidades:
      • Fuertes.
      • Débiles:
        • Dependencia en existencia.
        • Dependencia en identificación.
    • Atributos:
      • Identificativos.
      • Descriptivos.
      • Otros: derivados, discriminantes, multivaluados y compuestos.
    • Relaciones:
      • Grados: unaria (o reflexiva), binaria, doble, ternaria, n-aria.
      • Cardinalidades:
        • Uno a uno (1:1):
          • Participación (0, 1) - (0, 1).
          • Participación (0, 1) - (1, 1).
          • Participación (1, 1) - (0, 1).
          • Participación (1, 1) - (1, 1).
        • Uno a varios (1:N)
          • Participación (0, n) - (0, n)
          • Participación (0, n) - (1, n)
          • Participación (1, n) - (0, n)
          • Participación (1, n) - (1, n)
        • Varios a varios (N:M)
          • Participación (n, m) - (n, m).
      • Atributos en una relación.
      • Relaciones de dependencia: en existencia y en identificaciónn.

  3. Modelo E-R Extendido:
    • Restricciones en las relaciones:
      • De exclusividad.
      • De exclusión.
      • De inclusividad.
      • De inclusión.
    • Relaciones de jerarquía:
      • Exclusiva y total.
      • Exclusiva y parcial.
      • Solapada y total.
      • Solapada y parcial.
    • Agregaciones:
      • Compuesto/componente.
      • Miembro/Colección.

  4. Modelo Relacional:
    • Relación (o tabla):
      • Columnas (o atributos o campos).
      • Filas (o registros o tuplas).
    • Claves:
      • Primaria (o principal).
      • Secundaria (o foránea).
    • Relaciones.

  5. Transformación de un Modelo E-R a un Modelo Relacional:
    • Transformación de entidades: cada entidad se transforma en una tabla.
    • Transformación de atributos.
    • Transformación de relaciones:
      • Relaciones 1:1:
        • Caso 1: si las dos entidades participan con participación (0,1), entonces se crea una nueva tabla para la relación.
        • Caso 2: si alguna entidad, pero no las dos, participa con participación mínima 0 (0,1), entonces se pone la clave ajena en dicha entidad, para evitar en lo posible, los valores nulos.
        • Caso 3: si tenemos una relación 1:1 y ninguna tiene participación mínima 0, elegimos la clave principal de una de ellas y la introducimos como clave clave ajena en la otra tabla. Se elegirá una u otra forma en función de como se quiera organizar la información para facilitar las consultas. Los atributos propios de la relación pasan a la tabla donde se introduce la clave ajena.

  6. Normalización:
    • Primera Forma Normal (1FN).
    • Segunda Forma Normal (2FN).
    • Tercera Forma Normal (3FN).
    • Forma Normal de Boyce-Codd (FNBC).
    • Cuarta Forma Normal (4FN).
    • Quinta Forma Normal (5FN).
