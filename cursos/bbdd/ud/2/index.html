2. Modelo Entidad-Relación y Modelo Relacional

Contenidos

  1. Creación de una base de datos:
    • Análisis:
      • Especificación de Requisitos de Software (ERS).
    • Diseño:
      • Fase 1 - Conceptual: modelo de datos Modelo Entidad-Relación (E-R).
      • Fase 2 - Lógico: modelo de base de datos Modelo Relacional.
      • Fase 3 - Físico: modelo físico (implementado en SQL) (se ve en la próxima unidad).
  2. Modelo Entidad-Relación:
    • Entidades:
      • Fuertes.
      • Débiles.
    • Atributos.
      • (identificativos, descriptivos, derivados, discriminantes, multivaluados, compuestos)
    • Relaciones:
      • Grados: unaria (o reflexiva), binaria, doble, ternaria y n-aria.
      • Cardinalidades:
        • Uno a uno (1:1).
        • Uno a varios (1:N).
        • Varios a varios (N:M)
  3. Modelo Relacional:
    • Relación (o tabla):
      • Columnas (o atributos o campos).
      • Filas (o registros o tuplas).
    • Claves:
      • Primaria (o principal).
      • Secundaria (o foránea).
    • Relaciones.
  4. Transformación de un Modelo E-R a un Modelo Relacional:
    • Entidades.
    • Atributos.
    • Relaciones.
  5. Normalización:
    • Primera Forma Normal (1FN).
    • Segunda Forma Normal (2FN).
    • Tercera Forma Normal (3FN).
    • Forma Normal de Boyce-Codd (FNBC).
    • Cuarta Forma Normal (4FN).
    • Quinta Forma Normal (5FN).
  6. Material de apoyo: Gestión de Bases de Datos (teoría, actividades), José Juan Sánchez.
