• Todos los documentos tienen que estar validados en XHTML 5.2, WCAG 2.0 AAA y CSS3.

PRÁCTICA

Crea una web dinámica (XHTML + CSS + PHP) que mediante conexiones a una base de datos PosgreSQL proporcione un servicio y genere tanto un documento HTML final como un PDF final.

https://www.ibm.com/developerworks/library/os-php-twitter-interface/index.html

  • Inicio de sesión tanto como usuario como administrador
  • Registro como usuario inactivo, el administrador será el que le active.
  • Generar un informe tanto en HTML (más CSS) como en PDF recogiendo la información de una base de datos (PosgreSQL).
  • Cerrar sesión tanto como usuario como administrador.

Elige uno de los siguientes temas:

Sistema de test online

El usuario iniciará sesión y realizará una prueba online tipo test sobre PHP (mínimo 10 preguntas). El profesor, con rol administrador, iniciará sesión y podrá comprobar la nota de cada alumno y podrá generar un informe tanto en HTML (más CSS) como en PDF. Dicho informe mostrará las notas de cada alumno y estadísticas como media, moda, varianza, desviación típica, etc.

Informe de salud personal

El usuario introducirá su edad, altura, peso, medidas, actividad y todos los datos necesarios. La aplicación web generará un informe de salud tanto en HTML (incluir el estilo dentro del archivo HTML) como en PDF. Un ejemplo puede encontrarse aquí.

Clon simple de Twitter

El usuario iniciará sesión y podrá: escribir mensajes, ver mensajes (tanto suyos como de sus amigos), realizar peticiones de amistad y aceptar/rechazar peticiones de amistad. El administrador iniciará sesión y podrá ver los mensajes de cada usuario y podrá generar un informe tanto en HTML (más CSS) como en PDF. Dicho informe mostrará el usuario con más mensajes, el usuario con menos mensajes, el usuario con más amigos, el usuario con menos amigos y otras estadísticas como, por ejemplo, media de mensajes al día.
